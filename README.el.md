# 🌍 Nightwatch — ειδοποιήσεις σεισμών για την Ελλάδα

[🇬🇧 English](README.md) · **🇬🇷 Ελληνικά**

[![Earthquake monitor](https://github.com/F0rgiv3n/nightwatch/actions/workflows/monitor.yml/badge.svg)](https://github.com/F0rgiv3n/nightwatch/actions/workflows/monitor.yml)
[![Python](https://img.shields.io/badge/python-3.11-blue)](https://www.python.org/)
[![License: MIT](https://img.shields.io/badge/license-MIT-green)](LICENSE)

Το Nightwatch παρακολουθεί τους σεισμούς στην Ελλάδα και στέλνει μια ειδοποίηση
στο κινητό μου λίγα λεπτά αφότου γίνει ένας. Είναι δωρεάν, τρέχει μόνο του, και
χρησιμοποιεί μόνο επίσημα, δημόσια δεδομένα.

Το έφτιαξα για έναν απλό λόγο: η Ελλάδα σείεται *πολύ* — μικροί και μέτριοι
σεισμοί γίνονται σχεδόν κάθε μέρα — και ήθελα μια ήσυχη, αξιόπιστη ειδοποίηση στο
κινητό, χωρίς να βάλω άλλη μια εφαρμογή γεμάτη διαφημίσεις ούτε να πληρώνω server
μόνο και μόνο για να κρατάει ζωντανό ένα script.

Είναι σκόπιμα μικρό project, αλλά κάνει όλο τον κύκλο από την αρχή ως το τέλος:
ρωτάει μια επίσημη πηγή για πρόσφατους σεισμούς → βρίσκει ποιοι είναι καινούριοι
→ στέλνει ειδοποίηση → θυμάται τι έχει ήδη στείλει → το επαναλαμβάνει με
χρονοπρογραμματισμό. Όλο το πρόγραμμα είναι ένα ευανάγνωστο αρχείο Python.

> ℹ️ **Είναι proof-of-concept / project για portfolio, όχι σύστημα ασφαλείας.**
> Δουλεύει αρκετά καλά ώστε να είναι πραγματικά χρήσιμο και να δείχνει το
> pipeline, αλλά μην βασίζεσαι σε αυτό ως επίσημο σύστημα έγκαιρης
> προειδοποίησης. Περισσότερα στις [*Σημειώσεις &
> περιορισμούς*](#σημειώσεις--περιορισμοί).

## Τι άλλαξα, και γιατί (Ιούνιος 2026)

Αυτό είναι το ενδιαφέρον κομμάτι, οπότε αξίζει να το πω ειλικρινά.

Αρχικά το Nightwatch έπαιρνε τα δεδομένα του από το **USGS** (την αμερικανική
γεωλογική υπηρεσία). Δούλευε — μέχρι που δεν δούλεψε. Ένας πραγματικός σεισμός
**M3.2 κοντά στην Κρήτη** πέρασε χωρίς καμία ειδοποίηση, και μετά, δύο μέρες
αργότερα, ένας μεγαλύτερος σεισμός *με ειδοποίησε* κανονικά. Αυτό είναι ανάποδο,
οπότε το έψαξα.

Το πρόβλημα ήταν η πηγή δεδομένων. Το USGS είναι **παγκόσμιος** κατάλογος: άριστο
για μεγάλα γεγονότα οπουδήποτε στη Γη, αλλά συστηματικά **χάνει μέτριους (M3–4)
ελληνικούς σεισμούς, ή τους καταγράφει με καθυστέρηση**. Ο σεισμός που έχασα απλώς
δεν υπήρχε στο feed του USGS εκείνη τη στιγμή — παρότι το ελληνικό εθνικό δίκτυο
τον είχε μέσα σε ένα λεπτό.

Έτσι άλλαξα την πηγή στο **Αστεροσκοπείο Αθηνών (NOA)** — το Γεωδυναμικό
Ινστιτούτο, το εθνικό σεισμικό δίκτυο της Ελλάδας (και αυτό που τροφοδοτεί το
ευρωπαϊκό EMSC). Καταγράφει τους ελληνικούς σεισμούς πολύ πληρέστερα και πολύ
ταχύτερα. Με την ευκαιρία:

- **κατέβασα το όριο ειδοποίησης από M3.5 σε M3.0** — το NOA δίνει τα μεγέθη στην
  τοπική κλίμακα **ML / Ρίχτερ**, ακριβώς αυτή που λένε τα ελληνικά ΜΜΕ·
- **άλλαξα τι ανοίγει η ειδοποίηση** — το NOA δεν έχει δημόσια σελίδα ανά σεισμό,
  οπότε πλέον με ένα tap ανοίγει το **επίκεντρο σε χάρτη** (με το μέγεθος, την
  περιοχή και το βάθος μέσα στην ίδια την ειδοποίηση).

Η παλιά πηγή USGS παραμένει στον κώδικα ως εναλλακτική, για την περίπτωση που
φανεί χρήσιμη σε άλλη περιοχή.

## Πώς δουλεύει

```
                 ┌──────────────────────────────────────────────┐
   GitHub Actions│  κάθε 5 λεπτά   →  python nightwatch.py       │
   (το "cron")   └───────────────────────┬──────────────────────┘
                                         │
                 ┌───────────────────────▼──────────────────────┐
                 │ 1. Ρώτα το NOA για πρόσφατους σεισμούς μέσα   │
                 │    στο κουτί της Ελλάδας, πάνω από το μέγεθος │
                 │ 2. Σύγκρινε με το seen.json (όσα ήδη έστειλα) │
                 │ 3. Για κάθε ΝΕΟ σεισμό → στείλε ntfy push     │
                 │ 4. Αποθήκευσε το ενημερωμένο seen.json        │
                 └───────────────────────┬──────────────────────┘
                                         │
                          ntfy.sh ───────▼──────►  📱 το κινητό μου
```

Μερικές λεπτομέρειες που το κάνουν να συμπεριφέρεται σωστά:

- Φιλτράρει με ένα **γεωγραφικό «κουτί»** γύρω από την Ελλάδα (ηπειρωτική χώρα,
  Αιγαίο, νησιά), ώστε να παίρνεις σεισμούς μόνο από την περιοχή που σε ενδιαφέρει.
- **Στην πρώτη εκτέλεση μένει σιωπηλό** — απλώς καταγράφει ως baseline ό,τι ήδη
  υπάρχει, για να μη σε πλημμυρίσει με ειδοποιήσεις στο ξεκίνημα.
- Θυμάται τι έχει αναφέρει στο `seen.json`, οπότε κάθε σεισμό τον παίρνεις **μία
  φορά**.
- Στο GitHub Actions αυτή η κατάσταση κρατιέται στο **cache του Actions** ανάμεσα
  στις εκτελέσεις, ώστε το bot να θυμάται τι έστειλε χωρίς να «λερώνει» το git με
  αυτόματα commits.

## Τι αγγίζει (για όσους κοιτούν το portfolio)

Μικρό, αλλά περνάει από μια ρεαλιστική, ολοκληρωμένη αλυσίδα εργαλείων:

| Τομέας | Τι δείχνει |
| --- | --- |
| Ενσωμάτωση δημόσιου API | Κατανάλωση του REST API `fdsnws` του NOA και parsing της μορφής FDSN text |
| Γεωχωρικό φιλτράρισμα | Ερώτημα με bounding box (lat/lon) για μια περιοχή |
| Εντοπισμός αλλαγών | Λογική «τι είναι νέο από την προηγούμενη φορά» με κατάσταση |
| Push notifications | Ειδοποιήσεις βασισμένες σε συμβάντα, μέσω ntfy |
| Containerization | Ένα μικρό, single-stage Dockerfile |
| CI/CD & αυτοματισμός | Προγραμματισμένο GitHub Actions workflow που τρέχει 24/7 |
| Διαχείριση secrets | Ο στόχος ειδοποίησης ζει σε GitHub Actions secret, ποτέ hard-coded |
| Stateful workflows | Διατήρηση του `seen.json` ανάμεσα στις εκτελέσεις μέσω του Actions cache |

## Πώς να το τρέξεις

### Τοπικά

```bash
pip install -r requirements.txt

export NTFY_TOPIC=to-diko-sou-monadiko-topic     # διάλεξε όποιο μοναδικό όνομα θες
python nightwatch.py
```

Κάνε Subscribe στο ίδιο topic μέσα από την εφαρμογή [ntfy](https://ntfy.sh) (ή
άνοιξε `https://ntfy.sh/to-diko-sou-monadiko-topic` στον browser) για να
λαμβάνεις τις ειδοποιήσεις.

### Με Docker

```bash
docker build -t nightwatch .
docker run -e NTFY_TOPIC=to-diko-sou-monadiko-topic nightwatch
```

### 24/7 με GitHub Actions (δωρεάν)

1. Κάνε fork/clone το repo στον δικό σου λογαριασμό GitHub.
2. Πρόσθεσε ένα **secret** με όνομα `NTFY_TOPIC`
   (Settings → Secrets and variables → Actions) με την τιμή του topic σου.
3. Τέλος — το workflow στο [`.github/workflows/monitor.yml`](.github/workflows/monitor.yml)
   τρέχει κάθε 5 λεπτά, και μπορείς να το τρέξεις και χειροκίνητα από το tab
   **Actions** («Run workflow»).

Η πρώτη εκτέλεση καταγράφει ένα σιωπηλό baseline· οι ειδοποιήσεις ξεκινούν από
τον επόμενο νέο σεισμό.

## Παραμετροποίηση

Όλα βρίσκονται στο [`config.yaml`](config.yaml):

```yaml
interval: 300   # δευτερόλεπτα ανάμεσα στους ελέγχους (όταν τρέχει ως συνεχής διεργασία)

sources:
  - name: greece-earthquakes-noa
    type: noa               # Αστεροσκοπείο Αθηνών (η καλύτερη πηγή για Ελλάδα)
    min_magnitude: 3.0      # ειδοποίηση μόνο για μέγεθος 3.0 και πάνω (Ρίχτερ / ML)
    period_days: 1          # κοίτα πόσες μέρες πίσω σε κάθε έλεγχο

    # Bounding box γύρω από την Ελλάδα (latitude / longitude):
    min_latitude: 34.0
    max_latitude: 42.0
    min_longitude: 19.0
    max_longitude: 29.5
```

| Ρύθμιση | Σημασία |
| --- | --- |
| `min_magnitude` | Αγνόησε σεισμούς μικρότερους από αυτό |
| `period_days` | Πόσες μέρες πίσω να κοιτάει σε κάθε έλεγχο |
| `min/max_latitude`, `min/max_longitude` | Το «κουτί» (περιοχή) προς παρακολούθηση |
| `interval` | Δευτερόλεπτα ανάμεσα στους ελέγχους (μόνο στον συνεχή βρόχο, όχι στο Actions) |

Θες άλλη περιοχή; Άλλαξε το bounding box. Πολλές ειδοποιήσεις; Ανέβασε το
`min_magnitude` — η Ελλάδα έχει όντως έντονη σεισμικότητα.

## Προσθήκη άλλης πηγής

Κάθε πηγή είναι απλώς μια συνάρτηση που επιστρέφει λίστα από
`{"id", "title", "url"}`. Την καταχωρείς στο dictionary `SOURCES` μέσα στο
`nightwatch.py` και αναφέρεις τον `type` της στο `config.yaml` — αυτό είναι όλο
το «συμβόλαιο». Μια πηγή `usgs` υπάρχει δίπλα στο `noa` ως έτοιμο παράδειγμα.
Φυσικές επόμενες προσθήκες: το EMSC, ειδοποιήσεις καιρού, ή οτιδήποτε έχει
δημόσιο API.

## Σημειώσεις & περιορισμοί

- **Είναι proof of concept.** Project για portfolio/demo, όχι «σκληραγωγημένη»
  υπηρεσία. Η κάλυψη, η αξιοπιστία και ο χρόνος είναι αρκετά καλά για να δείξουν
  το pipeline να δουλεύει — όχι κάτι για να ρισκάρεις την ασφάλειά σου.
- **Για τον χρόνο.** Πάνω στο GitHub Actions *δεν* είναι real-time. Το cron του
  GitHub τρέχει το πολύ κάθε ~5 λεπτά και συχνά καθυστερεί, οπότε δες τις
  ειδοποιήσεις ως «μέσα σε μερικά λεπτά». Για πραγματικά γρήγορη, αξιόπιστη
  παράδοση, τρέξε το **Docker** container συνεχώς σε έναν μικρό cloud host (ή σε
  οποιοδήποτε always-on μηχάνημα) — τρέχει στον δικό του μικρό βρόχο `interval`
  αντί να περιμένει τον χρονοπρογραμματιστή του GitHub.
- **Πηγή δεδομένων.** Το NOA (Αστεροσκοπείο Αθηνών) είναι το εθνικό δίκτυο της
  Ελλάδας: καταγράφει τους ελληνικούς σεισμούς πολύ πληρέστερα και ταχύτερα από
  το παγκόσμιο feed της USGS, και δίνει τα μεγέθη στην τοπική κλίμακα ML / Ρίχτερ.
  Κάποια γεγονότα έρχονται ως γρήγορες αυτόματες λύσεις που μπορεί αργότερα να
  αναθεωρηθούν — αυτό είναι το τίμημα του real-time.
- **Τα ntfy topics** στον δημόσιο server είναι αναγνώσιμα από όποιον ξέρει το
  όνομα, οπότε διάλεξε ένα που δεν μαντεύεται εύκολα.
